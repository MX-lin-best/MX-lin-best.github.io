谷粒商城遇到的问题以及解决方法

本文最后更新于：2025年2月24日 下午

1、如有条件电脑内存至少16G起步，条件进一步加个屏幕，条件更进一步租一台至少4G内存的X86架构云服务器，所有部署的东西全扔云服务器上

2、P16，P17没法搭起来的建议照着rerenfast的github上的教程搭

3、项目搭起来后，每学完一章一定要用git提交一次，不然后续排错会令人崩溃

4、多记笔记，尤其是分布式，架构，锁相关的知识，有利于面试 5、IDEA的jrebel插件和ResetfulTool插件至少能提高一半的学习效率，必用

6、这章实在找不到解决办法看下弹幕，其余情况一律关闭

7、Maven依赖版本号一定要一模一样（除非你知道高版本怎么配置）

8、当前章节没达到老师的效果不要继续往下做，这个项目是线性的，你卡在这个环节后面基本也很难做下去，直到P325

9、单纯想学后端东西的，前端代码照抄，P28-P44跳过

10、ES的内存分配不要128MB，要256MB起步，商品上架的测试数据建议和老师的一模一样（除了图片），不然不好对照

11、nginx可以不用部署在服务器上，有windows版可以本地运行，跳过穿透

12、单点登录有条件直接用github的，不要用微博的，提高效率，可以用api.github.com/user这个接口

13、支付宝沙箱每周日12:00至每周一12:00维护不可用

14、个人认为新增商品，商品上架，保存订单数据，支付这四个功能为该项目最折磨人功能，一P能卡一两天

15、P100之前的内容属于当前JAVA后端能找到工作的最基本水平

16、调试环节是必要的，一定要跟着学，不会调试通过试用期堪忧

17、实际大多数公司的项目技术难点和涵盖范围不如这个项目（所以好多人边上班边学啊）

  4. Spring循环依赖问题

    • 解决方法

      1.添加@Lazy注解
  	对于类A和类B都是通过构造器注入的情况，可以在A或者B的构造函数的形参上加个@Lazy注解实现延迟加载。
    @Lazy实现原理是，当实例化对象时，如果发现参数或者属性有@Lazy注解修饰，那么就不直接创建所依赖的对象了，而是使用动态代理创建一个代理类
      2.Relying upon circular references is discouraged and they are prohibited by default. Update your application to remove the dependency cycle between beans. As a last resort, it may be possible to break the cycle automatically by setting spring.main.allow-circular-references to true.

根据提示信息在配置文件中添加

7. 关于**BeanUtils.copyProperties()**的**浅拷贝**问题**(无法拷贝集合中的不同包下的相同类名)**

   - 源码

      ```java
      /**
      	 * Copy the property values of the given source bean into the target bean.
      	 * Note: The source and target classes do not have to match or even be derived
      	 * from each other, as long as the properties match. Any bean properties that the
      	 * source bean exposes but the target bean does not will silently be ignored.
      	 * This is just a convenience method. For more complex transfer needs,
      	 * consider using a full BeanWrapper
      	 * As of Spring Framework 5.3, this method honors generic type information
      	 * when matching properties in the source and target objects.
      	 * The following table provides a non-exhaustive set of examples of source
      	 * and target property types that can be copied as well as source and target
      	 * property types that cannot be copied.
      	 * 
      	 *  source property type 		target property type 		copy supported 
      	 *  Integer 				    Integer 					yes 
      	 *  Integer 				    Number 						yes
      	 *  List<Integer> 				List<Integer> 				yes 
      	 *  List<?> 				    List<?> 					yes 
      	 *  List<Integer> 				List<?> 					yes 
      	 *  List<Integer> 				List<? extends Number> 		 yes 
      	 *  String 					    Integer 				    no 
      	 *  Number 					    Integer 					no 
      	 *  List<Integer>  				List<Long>} 				 no 
      	 *  List<Integer>				List<Number>}				 no
      	 * 
      	 * @param source the source bean
      	 * @param target the target bean
      	 * @throws BeansException if the copying failed
      	 * @see BeanWrapper
      	 */
      	public static void copyProperties(Object source, Object target) throws BeansException {
      		copyProperties(source, target, null, (String[]) null);
      	}

    • 解决(构造深拷贝工具类（类BeanUtils.copyProperties()）)

      package com.lexr.gulimall.product.utils;

import org.springframework.beans.BeanUtils;
import java.lang.reflect.*;
import java.util.*;

/**
 * @ClassName: DeepBeanUtils
 * @Author: li rui
 * @Date: 2025/2/16 15:42
 * @Description:
 */
public class DeepBeanUtils {

    /**
     * 深拷贝对象属性（包括集合和嵌套对象）
     * @param source 源对象
     * @param target 目标对象
     */
    public static void copyProperties(Object source, Object target) {
        if (source == null || target == null) return;
        copyPropertiesRecursive(source, target);
    }

    // 递归拷贝核心逻辑
    private static void copyPropertiesRecursive(Object source, Object target) {
        try {
            // 1. 先执行 Spring 的浅拷贝（处理基本类型和非集合属性）
            BeanUtils.copyProperties(source, target);

            // 2. 处理所有集合属性
            for (Field targetField : getAllFields(target.getClass())) {
                Field sourceField = getField(source.getClass(), targetField.getName());
                if (sourceField == null) continue;

                sourceField.setAccessible(true);
                targetField.setAccessible(true);

                Object sourceValue = sourceField.get(source);
                if (sourceValue == null) continue;

                // 仅处理 Collection 类型
                if (sourceValue instanceof Collection) {
                    handleCollection(sourceValue, targetField, target);
                }
            }
        } catch (Exception e) {
            throw new RuntimeException("深拷贝失败", e);
        }
    }

    // 处理集合类型
    private static void handleCollection(Object sourceCollection, Field targetField, Object target)
            throws Exception {

        Collection<?> sourceList = (Collection<?>) sourceCollection;
        if (sourceList.isEmpty()) return;

        // 获取目标集合的泛型类型
        Class<?> targetElementType = getGenericType(targetField);

        // 创建目标集合实例（支持 List/Set）
        Collection<Object> targetCollection = createTargetCollection(targetField.getType());

        // 遍历源集合元素，递归拷贝到目标类型
        for (Object sourceItem : sourceList) {
            Object targetItem = createAndCopy(sourceItem, targetElementType);
            if (targetItem != null) {
                targetCollection.add(targetItem);
            }
        }

        // 将新集合设置到目标对象
        targetField.set(target, targetCollection);
    }

    // 创建并拷贝嵌套对象
    private static Object createAndCopy(Object sourceItem, Class<?> targetType) {
        try {
            Object targetItem = targetType.getDeclaredConstructor().newInstance();
            copyPropertiesRecursive(sourceItem, targetItem);
            return targetItem;
        } catch (Exception e) {
            return null;
        }
    }

    // 获取字段的泛型类型（仅处理单参数化类型如 List<T>）
    private static Class<?> getGenericType(Field field) {
        ParameterizedType genericType = (ParameterizedType) field.getGenericType();
        return (Class<?>) genericType.getActualTypeArguments()[0];
    }

    // 创建目标集合实例
    private static Collection<Object> createTargetCollection(Class<?> collectionType) {
        if (List.class.isAssignableFrom(collectionType)) {
            return new ArrayList<>();
        } else if (Set.class.isAssignableFrom(collectionType)) {
            return new HashSet<>();
        }
        throw new IllegalArgumentException("不支持的集合类型: " + collectionType);
    }

    // 获取类及其父类的所有字段
    private static List<Field> getAllFields(Class<?> clazz) {
        List<Field> fields = new ArrayList<>();
        while (clazz != null) {
            fields.addAll(Arrays.asList(clazz.getDeclaredFields()));
            clazz = clazz.getSuperclass();
        }
        return fields;
    }

    // 安全获取字段（忽略不存在的字段）
    private static Field getField(Class<?> clazz, String name) {
        try {
            return clazz.getDeclaredField(name);
        } catch (NoSuchFieldException e) {
            return null;
        }
    }
}

有用技巧

  1. OpenFeign使用（远程调用其他服务）

    1. 引入依赖包

      <dependency>
           <groupId>org.springframework.cloud</groupId>
           <artifactId>spring-cloud-starter-openfeign</artifactId>
</dependency>

    2. 主启动类开启**@EnableFeignClients** 功能

    3. 编写接口， 加上注解**@FeignClient(要调用的服务名称)，接口中写要调用的Controller**中的方法，进行远程调用

      原理(调用的请求路径参数不同时，会匹配相对应的实体属性名。JSON相同，不需用实体类一样)：

      1.Controller中请求路径入参中添加了@RequestBody,会将对象转换为JSON，

      ​ 2.找到要调用的服务名，给请求路径发送请求，将JSON放在请求体位置，发送请求

      ​ 3.将请求体中的JSON转为调用服务中的实体

  2. Nacos作为配置中心

    1. 引入依赖

      <!--nacos-config依赖-->
<dependency>
      <groupId>com.alibaba.cloud</groupId>
      <artifactId>spring-cloud-starter-alibaba-nacos-config</artifactId>
</dependency>

    2. 创建一个bootstrap.properties

      spring.application.name=gulimall-coupon
spring.cloud.nacos.server-addr=127.0.0.1:8848

    3. 需要给配置中心添加一个叫 数据集 (Data ID) gulimall-coupon.properties.默认规则，应用名.properties

    4. 给应用名.properties添加任何配置

    5. 动态获取配置

      • @RefreshScope：动态获取并刷新配置

      • @Value（”${配置项的名}”）：获取配置

        如果配置中心和当前应用的配置文件都配置了相同的项，优先使用配置中心的配置

  3. JSR303校验使用

    • 在Controller中的请求路径入参中添加**@Valid注解，在实体类上标注注解@NotNull等，入参中添加BindingResult**可以获取响应结果
