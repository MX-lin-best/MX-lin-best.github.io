简介

Spring Boot是由Pivotal团队提供的全新框架，其设计目的是用来简化新Spring应用的初始搭建以及开发过程。该框架使用了特定的方式来进行配置，从而使开发人员不再需要定义样板化的配置。通过这种方式，Spring Boot致力于在蓬勃发展的快速应用开发领域(rapid application development)成为领导者。

参考笔记：SpringBoot2核心技术与响应式编程

官网文档：Spring Boot

四大核心

  1. 自动配置 -Auto Configuration
  2. 起步依赖-Starter Dependency
  3. 命令行界面-Spring Boot CLI
  4. 运行监控-Actuator

自定义配置映射到对象

  1. application.properties

    school.name=lexrSchool

    Controller

    @Controller
public class IndexController {

    @Value("${school.name}")
    private String schoolName;

    @RequestMapping("/say")
    public @ResponseBody
    String name() {
        return "Hello,SpringBoot" + schoolName;
    }


}

  2. application.properties

    school.name=lexrSchool

    pojo

    @Component
@ConfigurationProperties(prefix = "school")
public class School {

    private String name;

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }
}

    controller

    @Controller
public class IndexController {

    @Autowired
    private School school;

    @RequestMapping("/say")
    public @ResponseBody
    String name() {
        return "Hello,SpringBoot" + school.getName();
    }


}

SpringBoot集成JSP

  1. 引入依赖和编译的目标文件

    <!--引入SpringBoot内嵌Tomcat对jsp的解析依赖，不添加解析不了-->
<dependency>
    <groupId>org.apache.tomcat.embed</groupId>
    <artifactId>tomcat-embed-jasper</artifactId>
</dependency>
    <build>
    <resources>
        <resource>
            <directory>src/main/webapp</directory>
            <targetPath>META-INF/resources</targetPath>
            <includes>
                <include>*.*</include>
            </includes>
        </resource>
    </resources>
    ...
</build>

  2. SpringBoot默认的前端视图解析器是thymeleaf,application.properties中配置前后缀

    spring.mvc.view.prefix=/
spring.mvc.view.suffix=.jsp

  3. 编写java代码，页面在webapp文件夹下say.jsp

    @Controller
public class IndexController {

    @RequestMapping("/say")
    public ModelAndView say() {
        ModelAndView mv = new ModelAndView();
        mv.addObject("message","Hello,SpringBoot");
        mv.setViewName("say");
        return mv;
    }


}

SpringBoot集成MyBatis

  1. 引入mysql和mybatis-spring-boot依赖
<dependency>
       <groupId>mysql</groupId>
       <artifactId>mysql-connector-java</artifactId>
   </dependency>
   
   <dependency>
       <groupId>org.mybatis.spring.boot</groupId>
       <artifactId>mybatis-spring-boot-starter</artifactId>
       <version>2.2.0</version>
   </dependency>

  1. 配置MyBatis逆向工程

    <!DOCTYPE generatorConfiguration PUBLIC
        "-//mybatis.org//DTD MyBatis Generator Configuration 1.0//EN"
        "http://mybatis.org/dtd/mybatis-generator-config_1_0.dtd">
<generatorConfiguration>
    <context id="simple" targetRuntime="MyBatis3Simple">
        <jdbcConnection driverClass="com.mysql.cj.jdbc.Driver"
                        connectionURL="jdbc:mysql://localhost:3306/mybatis?allowMultiQueries=true"
                        userId="root"
                        password="bestforyou2/"
        />

        <!--  -->
        <javaTypeResolver >
            <property name="forceBigDecimals" value="false" />
        </javaTypeResolver>

        <!-- javaModelGenerator：指定javaBean的生成策略
        targetPackage="test.model"：目标包名
        targetProject="\MBGTestProject\src"：目标工程
        -->
        <javaModelGenerator targetPackage="com.lexr.springboot.bean"
                            targetProject=".\src">
            <property name="enableSubPackages" value="true" />
            <property name="trimStrings" value="true" />
        </javaModelGenerator>

        <!-- sqlMapGenerator：sql映射生成策略： -->
        <sqlMapGenerator targetPackage="templates"
                         targetProject=".\src\resources">
            <property name="enableSubPackages" value="true" />
        </sqlMapGenerator>

        <!-- javaClientGenerator:指定mapper接口所在的位置 -->
        <javaClientGenerator type="XMLMAPPER" targetPackage="com.lexr.springboot.dao"
                             targetProject=".\src">
            <property name="enableSubPackages" value="true" />
        </javaClientGenerator>

        <!-- 指定要逆向分析哪些表：根据表要创建javaBean -->
        <table tableName="tbl_dept" domainObjectName="Department"></table>
        <table tableName="tbl_employee" domainObjectName="Employee"></table>
    </context>
</generatorConfiguration>

SpringBoot集成Redis

  1. 添加操作redis的数据类型的依赖

    <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-data-redis</artifactId>
</dependency>

  2. 配置文件开启连接redis服务器

    spring.redis.host=192.168.6.132
spring.redis.port=6379

  3. 写业务

    1. controller层

      @RequestMapping("/put")
    public @ResponseBody
    Object put(String key,String value){
        departmentService.put(key,value);
        return "值注入成功";
    }

    2. service层接口以及实现类

      public interface DepartmentService {
    void put(String key, String value);
}
      @Service
public class DepartmentServiceImpl implements DepartmentService {

    @Autowired
    private RedisTemplate<Object,Object> redisTemplate;

    @Override
    public void put(String key, String value) {
        redisTemplate.opsForValue().set(key,value);
    }
}

SpringBoot解决浏览器乱码问题

  • 第一种

    • 使用characterEncodingFilter,resp.setContentType("text/html;character=utf-8")，提前关闭springboot默认的字符编码设置

  • 第二种

    • springboot的配置文件中修改

      server.servlet.encoding.enabled=true
server.servlet.encoding.force=true
server.servlet.encoding.charset=utf-8

SrpingBoot程序部署war

  1. SpringBoot Web 项目

    • 创建SpringBoot web工程

    • 添加SpringBoot 解析jsp依赖

    • 在pom.xml文件中配置jsp文件解析目录

      <dependency>
            <groupId>org.apache.tomcat.embed</groupId>
            <artifactId>tomcat-embed-jasper</artifactId>
</dependency>

    • 创建webapp并指定为web资源文件夹

    • 在application.properties配置文件中添加jsp前后缀

    • 提供Controller方法分别返回字符串

    • 在webapp目录下编写jsp页面

    • 浏览器访问

  2. 打包war

    • 设置pom.xml文件的打包方式

      <packaging>war</packaging>

    • 在构建工程时设置资源目录源文件夹位置（重要）

      <resource>
    <directory>src/main/resources</directory>
    <includes>
        <include>**/*.*</include>
    </includes>
</resource>

    • 指定打包名称

      <finalName>springboot</finalName>

    • 将打包后的工程放入Tomcat文件夹下的webapps下

  3. 打开Tomcat服务器，网页访问网址（加上上下文路径，也就是你的fileName）

SpringBoot程序部署jar

和war差不多，只不过是已经内嵌有Tomcat服务器，直接命令java -jar xxx.jar就行了，其中项目中的端口号和上下文根就是application.properties的设置的值
